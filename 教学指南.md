## 1. 互联网是怎么工作的？

### 🏠 打个比方：你去餐厅吃饭

| 现实世界 | Web 世界 |
|----------|----------|
| 你（顾客） | **浏览器**（Chrome / Edge） |
| 餐厅地址 | **IP 地址**（如 `175.178.122.124`） |
| 菜单 | **网页 URL**（如 `/login`） |
| 服务员 | **Web 服务器**（Node.js + Express） |
| 厨房 | **数据库**（MySQL） |
| 你点的菜 | **HTTP 请求** |
| 端上来的菜 | **HTTP 响应**（HTML 页面） |

整个流程：

```
你输入网址 → 浏览器发送 HTTP 请求 → 服务器收到请求
→ 服务器查数据库（如需）→ 拼接 HTML → 返回给浏览器
→ 浏览器渲染成你看到的页面
```

### 🔢 IP 和端口 —— 餐厅的"门牌号"

- **IP 地址** 就像餐厅的街道地址：`175.178.122.124`
- **端口** 就像餐厅的多个入口：
  - 80 号门 → 网页（HTTP）
  - 443 号门 → 加密网页（HTTPS）
  - 3000 号门 → 你的 Node.js 程序

你的项目里用了 **iptables 端口转发**，把 80 端口收到的请求转给 3000 端口：

```
用户 → 80 端口 → iptables → 3000 端口 → 你的 Node.js 程序
```

### 📝 你项目里的代码对应

```javascript
// app.js 第 4-5 行
const PORT = 3000;
// ...
app.listen(PORT, () => {
    console.log(`学生选课管理系统已启动：http://localhost:${PORT}`);
});
```

> 💡 `app.listen(3000)` 就是告诉程序："在 3000 号门口等着，有人来就接待"

## 2. Node.js 是什么？

### 🐍 你熟悉的 Python vs Node.js

| 特性 | Python | Node.js |
|------|--------|---------|
| 语言 | Python | JavaScript |
| 运行方式 | `python hello.py` | `node hello.js` |
| 安装包 | `pip install xxx` | `npm install xxx` |
| 包管理文件 | `requirements.txt` | `package.json` |
| 导入模块 | `import xxx` | `require('xxx')` |

**核心区别：Node.js 自带一个 Web 服务器能力**。Python 纯语言本身做不到，需要用 Flask/Django 等框架。Node.js 天生就是为网络而生的。

### 📦 npm —— Node 的"应用商店"

```bash
npm install express    # 安装 express框架
npm install mysql2     # 安装 MySQL 驱动
npm install chart.js   # 安装 图表库
```

安装后的包放在 `node_modules/` 文件夹（这就是为什么  `.gitignore` 要排除它——太大了）。

`package.json` 记录了你依赖的所有包名和版本，别人拿到这个文件后运行 `npm install` 就能自动装好一切。

### 🔍 你项目里是怎么用的

```javascript
// app.js 第 1-5 行
const express = require('express');      // 引入 Express 框架
const session = require('express-session'); // 引入 Session 管理
const mysql = require('mysql2/promise');    // 引入 MySQL 连接池
const path = require('path');               // 引入路径工具
const dbConfig = require('./config');        // 引入自己的数据库配置
```

> 💡 `require()` 就是 JavaScript 版的 `import`。括号里写包名或文件路径。

## 3. Express 框架

### 🚦 交通指挥的比喻

如果把网站比作一座城市：
- **路由（Route）** = 道路，决定请求去哪
- **中间件（Middleware）** = 收费站/安检，拦截检查请求
- **请求（Request）** = 车辆
- **响应（Response）** = 目的地

### 🛣️ 路由 —— 网站的"地图"

Express 路由就是：**"当用户访问某个 URL 时，执行某段代码"**

```javascript
// 语法：app.方法('路径', 中间件, 处理函数)

// 当有人 GET 请求 /login 时，返回登录页面
app.get('/login', (req, res) => {
    res.render('login', { error: null });
});

// 当有人 POST 请求 /login 时（提交表单），处理登录逻辑
app.post('/login', async (req, res) => {
    const { username, password, role } = req.body;  // 从表单取值
    // 查数据库验证...
    res.redirect('/');  // 重定向到首页
});
```

**GET vs POST 的区别：**

| | GET | POST |
|------|-----|------|
| 比喻 | 在餐厅"看菜单" | 在餐厅"下单" |
| 数据位置 | URL 后面可见 | 请求体里隐藏 |
| 用途 | 查看页面、搜索 | 登录、提交表单 |
| 安全性 | 密码会暴露在 URL | 密码不会暴露 |

### 🔗 你项目的路由清单

```
GET  /login            → 登录页
POST /login            → 处理登录
GET  /                 → 首页仪表盘
GET  /college          → 学院列表
POST /college/add      → 添加学院
POST /college/edit/:id → 编辑学院
POST /college/delete/:id → 删除学院
GET  /enroll           → 选课页面
POST /enroll/add       → 执行选课
POST /enroll/drop      → 退课
GET  /my-grades        → 学生成绩单
GET  /audit-log        → 管理员查看操作日志
// ... 还有很多
```

注意 URL 里的 `:id`：
```
/college/edit/01   → 编辑编号为 01 的学院
/college/edit/02   → 编辑编号为 02 的学院
```
`:id` 是一个**占位符**（参数），通过 `req.params.id` 获取实际值。

### 🛂 中间件 —— "安检关卡"

请求到达路由处理函数之前，会先经过一串中间件：

```javascript
// app.js

// 1. 解析静态文件请求（CSS/图片等）
app.use(express.static('public'));

// 2. 解析 POST 表单数据（把表单变成 req.body 对象）
app.use(express.urlencoded({ extended: true }));

// 3. 解析 JSON 数据
app.use(express.json());

// 4. Session 管理（记住登录状态）
app.use(session({ secret: '...', resave: false, saveUninitialized: false }));

// 5. 自定义中间件：检查是否登录
function requireAuth(req, res, next) {
    if (req.session.user) return next();  // 已登录，放行
    res.redirect('/login');                // 未登录，踢回登录页
}

// 6. 自定义中间件：只允许管理员
function adminOnly(req, res, next) {
    if (req.session.user?.role_type === 'admin') return next();
    res.status(403).send('权限不足');
}
```

> 💡 `next()` 的意思是 "检查通过，交给下一个中间件继续处理"  
> 不调用 `next()` 就是 "拦住，不让过"

## 4. MySQL 数据库

### 🏗️ 从 Excel 到数据库

假设你用 Excel 存学生信息：

| 学号 | 姓名 | 性别 | 学院 |
|------|------|------|------|
| 20220101 | 张三 | 男 | 计算机学院 |
| 20220102 | 李四 | 女 | 数学学院 |

这就是一张**表（Table）**。每行是一条**记录（Row）**，每列是一个**字段（Column）**。

数据库就是很多张这样的表，加上它们之间的关系。

### 🗝️ 主键和外键 —— 表的"身份证"和"关系线"

**主键（Primary Key）**：每条记录的唯一标识，像身份证号一样，不允许重复。

```
t_student 表：f_stu_id（学号）是主键
t_course  表：f_course_id（课程编号）是主键
```

**外键（Foreign Key）**：建立表与表之间的关系。

```
学生选课表 t_stu_course：
  f_stu_id    → 引用 t_student 表（哪个学生选的）
  f_course_id → 引用 t_course 表（选的哪门课）
```

这样一来，就能**关联查询**——查出"张三选了哪些课"。

### 📊 你项目的 7 张核心表

```
t_admin          ← 管理员
t_college        ← 学院
t_teacher        ← 教师（外键→学院）
t_student        ← 学生（外键→学院）
t_course         ← 课程
t_teach_course   ← 授课关系（外键→教师 + 外键→课程）
t_stu_course     ← 选课关系（外键→学生 + 外键→授课）
t_operation_log  ← 操作日志
```

画成图：
```
  学院 ←── 教师 ←── 授课表 ──→ 课程
   │                         ↓
   └──→ 学生 ──→ 选课表 ←────┘
```

### 🔤 SQL 基础 —— 数据库的"语言"

SQL 就四种核心操作，记住缩写 **CRUD**：

#### Create（增）—— INSERT
```sql
INSERT INTO t_student (f_stu_id, f_name, f_sex)
VALUES ('20220103', '王五', '男');
```
```javascript
// 在 Node.js 里这样写：
await pool.query(
    'INSERT INTO t_student (f_stu_id, f_name, f_sex) VALUES (?, ?, ?)',
    ['20220103', '王五', '男']
);
```
> 💡 `?` 是**参数占位符**，防止 SQL 注入攻击（后面安全章节会讲）

#### Read（查）—— SELECT
```sql
-- 查所有学生
SELECT * FROM t_student;

-- 查计算机学院的所有学生
SELECT * FROM t_student WHERE f_college_id = '01';

-- 查学生 + 学院名称（联表查询）
SELECT s.f_name, c.f_name AS college_name
FROM t_student s
JOIN t_college c ON s.f_college_id = c.f_college_id;
```

**联表查询 JOIN 图解：**
```
t_student                  t_college
┌──────┬──────┬──────────┐  ┌──────┬──────────┐
│ 姓名 │ ...  │ 学院编号  │  │ 编号  │ 学院名称  │
├──────┼──────┼──────────┤  ├──────┼──────────┤
│ 张三 │ ...  │   01     │──│  01  │ 计算机    │
│ 李四 │ ...  │   02     │──│  02  │ 数学      │
└──────┴──────┴──────────┘  └──────┴──────────┘
        外键 ──────────────→ 主键
```

结果：
```
张三 | 计算机学院
李四 | 数学学院
```

#### Update（改）—— UPDATE
```sql
UPDATE t_student SET f_name = '张三丰' WHERE f_stu_id = '20220101';
```
> ⚠️ **千万别忘 `WHERE`！** `UPDATE t_student SET f_name = '张三丰'`（没有 WHERE）会把所有学生名字都改掉！

#### Delete（删）—— DELETE
```sql
DELETE FROM t_student WHERE f_stu_id = '20220103';
```
> ⚠️ 同理，千万别忘 `WHERE`！

### 📄 分页 —— 数据太多怎么办

如果表里有 10000 条记录，一次全拿出来页面会卡死。

**LIMIT + OFFSET** 解决：
```sql
-- 第 1 页：取前 10 条
SELECT * FROM t_student LIMIT 10 OFFSET 0;
-- 第 2 页：跳过前 10 条，再取 10 条
SELECT * FROM t_student LIMIT 10 OFFSET 10;
-- 第 3 页：跳过前 20 条，再取 10 条
SELECT * FROM t_student LIMIT 10 OFFSET 20;
```

你项目的实现：
```javascript
// app.js 第 9-16 行
const PER_PAGE = 10;  // 每页 10 条

function pages(req) {
    const p = Math.max(1, parseInt(req.query.page) || 1);
    return {
        page: p,
        offset: (p - 1) * PER_PAGE,  // 跳过前面的
        limit: PER_PAGE               // 每次取多少
    };
}
```

### 🔍 LIKE 模糊搜索

```sql
-- 搜索名字包含"张"的学生
SELECT * FROM t_student WHERE f_name LIKE '%张%';
```
- `%` 是通配符，表示"任意字符"
- `%张` = 以"张"结尾
- `张%` = 以"张"开头
- `%张%` = 包含"张"

你项目的搜索工具函数：
```javascript
// app.js 第 19-25 行
function searchWhere(req, fields) {
    const q = (req.query.q || '').trim();
    if (!q) return { where: '', params: [], q: '' };
    const clauses = fields.map(f => `${f} LIKE ?`);
    const params = fields.map(() => `%${q}%`);
    return { where: ' WHERE ' + clauses.join(' OR '), params, q };
}
// 结果：WHERE f_name LIKE ? OR f_stu_id LIKE ? OR ...
```

## 5. EJS 模板引擎

### 🎭 什么是模板引擎？

假设你开了 100 家连锁店，每家店门口都要挂"欢迎光临 XXX 分店"。你不会写 100 个不同的 HTML 文件，而是写一个模板：

```html
<!-- 模板 -->
<h1>欢迎光临 <%= 店名 %> 分店</h1>
```

然后程序自动填充：店名 = "北京"、"上海"、"广州"... 生成 100 个页面。

这就是 EJS 做的事：**把数据填进 HTML 模板，生成最终页面。**

### 📝 EJS 的三种标签

| 标签 | 含义 | 用于 |
|------|------|------|
| `<%= 变量 %>` | 输出，转义 HTML | 显示用户名、数据 |
| `<%- 变量 %>` | 输出，不转义 | 嵌入 HTML 片段 |
| `<% 代码 %>` | 执行 JS，不输出 | 循环、条件判断 |

### 🎯 实际例子：显示学生列表

```html
<!-- views/student.ejs 中的关键逻辑 -->

<!-- 条件判断：如果有数据就显示表格，否则显示空状态 -->
<% if (students.length === 0) { %>
    <p>暂无数据</p>
<% } else { %>

<!-- 循环：遍历每个学生，生成一行 -->
<% students.forEach(function(s) { %>
    <tr>
        <td><%= s.f_stu_id %></td>    <!-- 输出学号 -->
        <td><%= s.f_name %></td>      <!-- 输出姓名 -->
        <td><%= s.college_name %></td> <!-- 输出学院 -->
    </tr>
<% }) %>

<% } %>
```

假设 `students` 数据是：
```json
[
    { "f_stu_id": "001", "f_name": "张三", "college_name": "计算机" },
    { "f_stu_id": "002", "f_name": "李四", "college_name": "数学" }
]
```

生成的 HTML：
```html
<tr><td>001</td><td>张三</td><td>计算机</td></tr>
<tr><td>002</td><td>李四</td><td>数学</td></tr>
```

### incl()—— 模板复用

每个页面都有同样的侧边栏和底部，不需要在每个文件里复制粘贴：

```html
<!-- index.ejs 开头和结尾 -->
<%- include('header', { title: '首页' }) %>

<!-- 本页面的内容放在中间 -->

<%- include('footer') %>
```

`header.ejs` 包含 `<html><head>` 到侧边栏的所有 HTML。

`footer.ejs` 包含关闭标签、公共 JS 函数（搜索、Toast 通知等）。

## 6. Session 会话

### 🎫 你是谁？—— 登录状态的保持

**HTTP 是无状态的**：服务器处理完一个请求就"忘"了你。如果不做处理，每次刷新页面都要重新登录。

**Session（会话）** 就像游乐园的手环：
1. 你第一次来（登录），售票处验证身份后给你戴上手环
2. 之后每次玩项目，出示手环就行，不用再买票
3. 手环上有防伪标识（加密），别人伪造不了

你项目里的实现：

```javascript
// app.js
app.use(session({
    secret: 'scsmis-dev-key',   // 加密密钥，用于签名防伪
    resave: false,               // 不重复保存
    saveUninitialized: false     // 未登录不创建 Session
}));

// 登录成功 → 把用户信息存入 Session
req.session.user = {
    id: 'admin1',
    name: '张管理',
    role_type: 'admin'
};

// 后续请求 → 检查 Session 确认身份
function requireAuth(req, res, next) {
    if (req.session.user) return next();  // 有手环，放行
    res.redirect('/login');                // 没手环，去登录
}

// 退出登录 → 撕掉手环
app.get('/logout', (req, res) => {
    req.session.destroy();  // 销毁 Session
    res.redirect('/login');
});
```

### 👑 权限控制 —— 三个角色的不同权限

```javascript
// 管理员专用路由
app.get('/college', adminOnly, async (req, res) => {
    // 只有管理员能访问学院管理页面
});

function adminOnly(req, res, next) {
    if (req.session.user?.role_type === 'admin') return next();
    res.status(403).send('权限不足');  // 403 = 禁止访问
}
```

所有权限控制的模式：

```
路由 + 中间件 = 受保护的路由

/college       + adminOnly     = 只有管理员能看
/grade-entry   + requireAuth   = 登录的人都能看
/enroll/add    + requireAuth   = 登录了但还要校验学生身份
```

## 8. SQL 深入

### 👁️ 视图（View）—— 保存好的常用查询

视图就像一个**保存好的 SQL 查询**，当表来用：

```sql
-- 定义视图（在 scsmis.sql 里）
CREATE VIEW v_teacher_course AS
SELECT
    c.f_course_id AS 课程编号,
    c.f_name AS 课程名称,
    t.f_teach_id AS 教师工号,
    t.f_name AS 教师姓名,
    t.f_title AS 职称,
    tc.f_time AS 授课时间,
    tc.f_location AS 授课地点
FROM t_teach_course tc
JOIN t_teacher t ON tc.f_teach_id = t.f_teach_id
JOIN t_course c ON tc.f_course_id = c.f_course_id;
```

之后使用时就很简单了：
```sql
-- 不用再写复杂的 JOIN，直接查视图！
SELECT * FROM v_teacher_course WHERE 教师工号 = 'T0001';
```

### ⚡ 触发器（Trigger）—— 自动执行的"守卫"

当某个操作发生时，自动触发一段逻辑。

```sql
-- 你项目里：选课前检查学分是否超过 100
CREATE TRIGGER trg_check_credit
BEFORE INSERT ON t_stu_course   -- 在插入选课记录之前
FOR EACH ROW
BEGIN
    DECLARE total INT;
    SELECT COALESCE(SUM(c.f_credit), 0) INTO total
    FROM t_stu_course sc
    JOIN t_course c ON sc.f_course_id = c.f_course_id
    WHERE sc.f_stu_id = NEW.f_stu_id;

    IF total + (SELECT f_credit FROM t_course WHERE f_course_id = NEW.f_course_id) >= 100 THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '学分已达100上限，无法选课';
    END IF;
END;
```

> 💡 `NEW` 代表即将插入的那条新记录。  
> `SIGNAL` 是手动抛出一个错误。

## 9. 安全

### 🔓 SQL 注入 —— 最经典的攻击

**危险写法（永远不要这样做）：**
```javascript
const id = req.body.id;
// 如果用户输入：' OR '1'='1
await pool.query(`SELECT * FROM t_user WHERE id = '${id}'`);
// 实际执行的 SQL：
// SELECT * FROM t_user WHERE id = '' OR '1'='1'
// 结果：返回所有用户！因为 '1'='1' 永远为真
```

**安全写法（参数化查询）：**
```javascript
await pool.query('SELECT * FROM t_user WHERE id = ?', [id]);
// 参数 ? 会被自动转义，恶意输入无效
```

你项目里全部使用了 `?` 参数化查询，✅ 安全。

### 🔐 密码存储 —— 明文 vs 加密

当前项目密码是明文 `123456`，**生产环境必须改**。

```
❌ 错误：存明文 "123456"
✅ 正确：存哈希值 "$2b$10$3euPcmQFCiblsZeEu5s7p.9MUZWg3..."
```

bcrypt 加密原理：
```
用户注册时：123456 → bcrypt → $2b$10$...存入数据库
用户登录时：输入密码 → bcrypt → 和数据库里的哈希比对
即使数据库泄露，攻击者也反推不出原始密码
```

### 🌐 .env 环境变量 —— 密码不上传到 GitHub

```
❌ config.js 里写：
   password: 'yxy062517...'    ← 上传 GitHub 后全世界可见

✅ .env 文件里写：
   DB_PASSWORD=yxy062517...     ← 在 .gitignore 中，不会上传
   config.js 用 process.env.DB_PASSWORD 读取
```

### 🔀 iptables —— 端口转发

默认网页端口是 80，但你的程序监听 3000。普通用户不能访问 3000 端口：

```bash
# 把 80 端口的流量转发到 3000
sudo iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 3000
```

现在用户访问 `http://175.178.122.124`（默认 80 端口）时，流量会自动转到你的 3000 端口程序。

## 🎯 总结：一个请求的完整旅程

以"管理员添加一个学院"为例，走一遍全流程：

```
1. 管理员在浏览器打开 /college 页面
   ↓
2. 浏览器发送 GET /college 请求到服务器
   ↓
3. Express 路由匹配到 app.get('/college', adminOnly, ...)
   ↓
4. adminOnly 中间件检查 Session → 是管理员 → 放行
   ↓
5. 路由函数执行：SELECT * FROM t_college LIMIT 10 OFFSET 0
   ↓
6. MySQL 返回数据：[{学院编号:'01', 名称:'计算机'}, ...]
   ↓
7. res.render('college', { colleges: rows, ... })
   ↓
8. EJS 引擎读取 views/college.ejs，把数据填入循环
   ↓
9. 生成完整 HTML，返回给浏览器
   ↓
10. 浏览器渲染表格

---

管理员填写表单，点击"添加"
   ↓
11. 浏览器发送 POST /college/add，表单数据在请求体里
   ↓
12. express.urlencoded 中间件解析表单 → req.body
   ↓
13. 路由函数执行：INSERT INTO t_college VALUES (...)
   ↓
14. logOp() 写入审计日志
   ↓
15. res.redirect('/college') → 回到列表页
```
